# Tamsi — AI Academic Co-Pilot

## Project Overview

Tamsi is a web application built specifically for FEU Institute of Technology students.

The goal is to eliminate manual GWA calculations by allowing students to upload screenshots of their SOLAR Grade Report. AI extracts the academic records, the application computes all academic metrics deterministically, and GPT explains the results in natural language.

The application is NOT a chatbot.

The application is an academic dashboard powered by AI.

---

# Primary Users

Current FEU Tech undergraduate students.

Assume every user owns an FEU ChatGPT Edu account.

Authentication for AI usage should use **ChatGPT OAuth** instead of OpenAI API keys whenever possible.

The objective is that users consume their own ChatGPT entitlement instead of my own API credits.

# IMPORTANT ARCHITECTURE

The LLM must NEVER compute academic values.

The LLM is ONLY responsible for

- reading screenshots
- extracting structured information
- generating explanations
- answering academic questions based on computed data

The backend is responsible for

- GWA computation
- cumulative GWA
- trimester GWA
- honors eligibility
- scholarship eligibility
- projections
- statistics

All calculations must be deterministic TypeScript code.

Never ask GPT to perform mathematical calculations.

# Workflow

Student logs in.

↓

Student connects ChatGPT account.

↓

Student uploads SOLAR screenshot.

↓

GPT Vision extracts structured JSON.

↓

User reviews extracted data.

↓

User confirms.

↓

Backend computes academic metrics.

↓

Dashboard renders.

↓

GPT receives computed metrics and generates plain-language advice.

# OCR Extraction

GPT should return ONLY JSON.

Example format

```json
{
  "terms": [
    {
      "term": "AY2425-T3",
      "courses": [
        {
          "code": "CCS0071",
          "title": "Algorithms",
          "units": 3,
          "midterm": 3.5,
          "final": null,
          "status": "in_progress"
        }
      ]
    }
  ]
}
```

No markdown.

No prose.

No explanations.

Only valid JSON.

# Engineering Principles

Always prefer deterministic code over AI.

AI should interpret.

Backend should calculate.

The application should continue functioning even if the AI provider changes in the future.

Design the AI layer behind an interface so providers can be swapped later (OpenAI API, ChatGPT OAuth, Gemini, Ollama).

Keep AI provider implementation isolated from business logic.
